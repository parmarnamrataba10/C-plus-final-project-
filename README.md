# Programs Included

## Q1. Rectangle Class

### Aim

Create a `Rectangle` class with private data members `length` and `width`, setter and getter functions, and a function to calculate the area.

### Concepts Used

* Class and Object
* Data Encapsulation
* Setter and Getter Functions
* Member Functions

### Output

Displays the area of the rectangle.

---

## Q2. Employee Class

### Aim

Create an `Employee` class with private members `name`, `salary`, and `designation`. Use setter and getter functions to store and display employee details.

### Concepts Used

* Encapsulation
* Setter and Getter Functions
* Object Creation

### Output

Displays employee name, salary, and designation.

---

## Q3. Inheritance Example

### Aim

Create a base class `Shape` and derive `Circle` and `Rectangle` classes. Calculate and display the area of both shapes.

### Concepts Used

* Single Inheritance
* Base Class
* Derived Class
* Function Reuse

### Output

Displays the color and area of the circle and rectangle.

---

## Q4. Virtual Function Example

### Aim

Implement runtime polymorphism using virtual functions. Create a base class `Shape` and override the `display()` function in `Circle` and `Rectangle` classes.

### Concepts Used

* Virtual Functions
* Runtime Polymorphism
* Function Overriding
* Dynamic Memory Allocation
* Base Class Pointer

### Output

Displays the details and area of both shapes using virtual functions.

---

## Q5. Abstract Class Example

### Aim

Create an abstract class `Animal` with pure virtual functions `sound()` and `move()`. Implement these functions in `Dog` and `Bird` classes.

### Concepts Used

* Abstract Class
* Pure Virtual Functions
* Runtime Polymorphism
* Function Overriding

### Output

Displays the sound and movement of the dog and bird.

# Conclusion

These programs demonstrate the core principles of Object-Oriented Programming in C++. They provide practical knowledge of creating classes and objects, implementing encapsulation through setter and getter functions, using inheritance for code reusability, achieving runtime polymorphism with virtual functions, and designing abstract classes with pure virtual functions.
